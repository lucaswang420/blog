# 🛠️ GCP 云服务器 Xshell 连接与安全隧道配置指南

> [!NOTE]
> 本指南详细介绍如何使用 SSH 密钥对通过 Xshell 安全连接 GCP 服务器，并配置隧道访问 OpenClaw 管理后台。

---

## 一、准备工作：生成 SSH 密钥对（公钥上传到服务器，私钥留在本地）

在 Xshell 中使用密钥连接比密码更安全，这也是 GCP 的首选方式。

### 操作步骤

| 步骤 | 操作 | 说明 |
|:---:|------|------|
| **1** | 打开 **Xshell**，点击菜单栏 **工具** → **用户密钥管理者** | - |
| **2** | 点击 **生成**，选择 RSA 算法，位长度选 `2048` 或 `4096` | 建议选择 4096 位更安全 |
| **3** | 点击 **下一步** 填写密钥名称和注释 | 密钥名称不重名即可，注释需要为用户名（如 `test`），密码可忽略 |
| **4** | 点击 **下一步** 直至完成 | 按向导完成密钥生成 |
| **5** | 选中生成的密钥，点击 **属性** → **公钥** 选项卡 | 查看公钥内容 |
| **6** | **复制** 框内的公钥内容 | 格式通常以 `ssh-rsa AAAAB3...` 开头 |
| **7** | 导出私钥 | 私钥需要私密保管，导出保存到本地 `$env:USERPROFILE\.ssh\` 目录下 |

或者使用系统自带的 `ssh-keygen` 工具生成：

```bash
ssh-keygen -t rsa -b 2048 -f $env:USERPROFILE\.ssh\gcp_key -C test
```

---

## 二、在 GCP 后台注入公钥

### 操作步骤

| 步骤 | 操作 | 说明 |
|:---:|------|------|
| **1** | 登录 **Google Cloud Console**，进入 **Compute Engine** → **VM 实例** | - |
| **2** | 点击你的实例名称（如 `gcloud-openclaw-4c16m`），点击上方 **修改** | - |
| **3** | 下拉找到 **SSH 密钥** 部分，点击 **添加项** | - |
| **4** | 粘贴刚才在 Xshell 复制的公钥 | - |
| **5** | ⚠️ **重要**：确保公钥末尾包含用户名（如 `test`） | 如果没有请手动添加 |
| **6** | 点击 **保存** | 完成公钥注入 |

> [!TIP]
> 公钥格式示例：`ssh-rsa AAAAB3NzaC1yc2E... username`

---

## 三、配置 Xshell 会话

### 3.1 新建会话

| 配置项 | 值 |
|--------|-----|
| **名称** | `GCP-OpenClaw-Server` (建议) |
| **主机 (Host)** | 你的 GCP 静态外部 IP |
| **协议** | SSH |
| **端口** | `22` |

### 3.2 身份验证配置

| 配置项 | 值 |
|--------|-----|
| **方法** | `Public Key` |
| **用户名** | 你的用户名（如 `test`） |
| **用户密钥** | 选择第一步生成的密钥 |

---

## 四、配置安全隧道 (SSH Tunneling)

> [!IMPORTANT]
> 由于 OpenClaw 的管理端口 `18789` 未在公网开放，**必须通过此步骤**才能在本地浏览器进行访问。

### 操作步骤

1. 在会话属性窗口，左侧导航选择：**连接** → **SSH** → **隧道**。
2. 点击 **添加**，填写以下信息：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **类型 (Type)** | `Local (Outgoing)` | 本地转发 |
| **源主机 (Source Host)** | `localhost` | 或留空 |
| **侦听端口 (Listening Port)** | `18789` | 本地监听端口 |
| **目标主机 (Destination Host)** | `localhost` | 服务器本地地址 |
| **目标端口 (Destination Port)** | `18789` | OpenClaw 管理端口 |

3. 点击 **确定** 保存配置。

---

## 五、连接与使用

### 5.1 建立连接

| 步骤 | 操作 |
|:---:|------|
| **1** | 点击 **连接** |
| **2** | 首次连接会弹出主机密钥确认，点击 **接受并保存** |
| **3** | 验证连接成功，应看到类似 `test@gcloud-openclaw-4c16m:~$` 的终端提示符 |

### 5.2 访问管理后台

> [!NOTE]
> **连接条件**：只要 Xshell 保持连接状态，安全隧道即处于打通状态。

访问地址：**`http://localhost:18789`**

---

## 🔐 架构师提示

> [!TIP]
> 以后如果你需要在移动端或者其他电脑访问，只需在那台机器上建立相同的 SSH 隧道即可，**无需修改 GCP 的防火墙规则**。这样能最大程度保护服务器安全。

### 安全优势

| 方式 | 安全性 | 推荐度 |
|------|--------|--------|
| 开放公网端口 | ❌ 低风险 | 不推荐 |
| SSH 隧道访问 | ✅ 高安全性 | **强烈推荐** |

---

> [!TIP]
> **配置完成后**，你可以安全地从本地浏览器管理 OpenClaw，无需担心服务器暴露在公网风险中。
